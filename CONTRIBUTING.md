## Contributing to Circuit Foundry

* Create a branch from `main`
* Use Conventional Commits
* Keep PRs small and focused
* Add or update docs when changing gameplay systems
* No large art assets in PRs without LFS

#### Commit style

* feat: new feature
* fix: bug fix
* chore: tooling
* docs: documentation
* refactor: code refactor
* perf: performance improvements
* test: tests

#### Code conventions

* C# naming: PascalCase for public, camelCase for private
* Avoid MonoBehaviours for pure logic, prefer plain C# classes
* No item physics on belts
